# PHASE 1.5: DISCOVER (Monorepo & Module Detection)

**Goal:** Определить структуру проекта — single module или monorepo. Построить карту модулей и выбрать стратегию анализа.

**When:** После VALIDATE, перед DETECT.

**Required state:** `state.validate.path`, `state.validate.mode`, `state.validate.source_file_count`

---

## 1.5.1 Module Manifest Scan

Найти все manifest-файлы, определяющие границы модулей:

```bash
# Go modules
find "$PROJECT_PATH" -name "go.mod" -not -path "*/vendor/*" -not -path "*/.git/*"

# Node.js packages
find "$PROJECT_PATH" -name "package.json" -not -path "*/node_modules/*" -not -path "*/.git/*"

# Rust crates
find "$PROJECT_PATH" -name "Cargo.toml" -not -path "*/.git/*"

# Java/Kotlin
find "$PROJECT_PATH" -name "pom.xml" -o -name "build.gradle" -o -name "build.gradle.kts" | grep -v ".git"

# Python
find "$PROJECT_PATH" -name "pyproject.toml" -o -name "setup.py" -o -name "setup.cfg" | grep -v ".git"
```

**Decision:**
```
manifest_count = count of found manifests
if manifest_count <= 1:
    is_monorepo = false
elif manifest_count > 1:
    is_monorepo = true
```

**Edge case:** Monorepo с единственным root manifest, но несколькими "сервисами" в `services/` или `cmd/`:
```bash
# Дополнительная проверка для Go:
# Считать cmd/ entries как потенциальные сервисы
CMD_COUNT=$(ls -d cmd/*/ 2>/dev/null | wc -l)
SERVICES_COUNT=$(ls -d services/*/ 2>/dev/null | wc -l)

if [ $CMD_COUNT -gt 3 ] || [ $SERVICES_COUNT -gt 1 ]; then
    is_monorepo = true  # Multi-service monolith or monorepo
fi
```

## 1.5.2 Module Classification

Для каждого найденного модуля определить тип:

```bash
for module_path in $MODULE_PATHS; do
    # Определить язык по manifest
    if [ -f "$module_path/go.mod" ]; then
        language="go"
    elif [ -f "$module_path/package.json" ]; then
        language="typescript"  # или javascript — уточнить по tsconfig.json
    elif [ -f "$module_path/Cargo.toml" ]; then
        language="rust"
    elif [ -f "$module_path/pom.xml" ] || [ -f "$module_path/build.gradle" ]; then
        language="java"
    elif [ -f "$module_path/pyproject.toml" ]; then
        language="python"
    fi

    # Определить тип модуля
    relative_path=$(realpath --relative-to="$PROJECT_PATH" "$module_path")
    case "$relative_path" in
        services/*|svc/*)       type="service" ;;
        cmd/*)                  type="service" ;;
        pkg/*|lib/*|shared/*)   type="library" ;;
        internal/*)             type="library" ;;
        frontend/*|web/*|app/*) type="app" ;;
        tools/*|scripts/*)      type="tool" ;;
        *)                      type="service" ;;  # default
    esac
done
```

## 1.5.3 Inter-Module Dependency Detection

Определить зависимости между модулями проекта:

```bash
# Go: Парсить go.mod каждого модуля, найти ссылки на другие модули проекта
ROOT_MODULE=$(head -1 go.mod | awk '{print $2}')  # e.g. "github.com/org/project"

for module_path in $MODULE_PATHS; do
    if [ -f "$module_path/go.mod" ]; then
        # Найти imports других внутренних модулей
        grep "$ROOT_MODULE" "$module_path/go.mod" | grep -v "^module" | awk '{print $1}'
    fi
done

# Node.js: workspace dependencies
if [ -f "package.json" ]; then
    # Проверить workspaces
    grep -A 10 '"workspaces"' package.json
fi

# Rust: workspace members
if grep -q "workspace" Cargo.toml 2>/dev/null; then
    grep -A 20 "\[workspace\]" Cargo.toml | grep "members"
fi
```

## 1.5.4 Root Module Detection

Определить "главный" модуль — точку входа проекта:

```
Criteria для root module:
1. Содержит main.go / main.py / index.ts (entry point)
2. Находится в корне проекта или cmd/
3. Зависит от наибольшего числа других модулей
4. Имеет HTTP/gRPC server setup
```

```bash
# Go: найти main package в корне
if [ -f "cmd/main.go" ] || find cmd/ -name "main.go" -maxdepth 2 2>/dev/null | head -1; then
    root_module="cmd/"
fi

# Или по наибольшему fan-out
# (модуль с наибольшим количеством internal imports)
```

## 1.5.5 Strategy Selection

На основе анализа выбрать стратегию:

| Condition | Strategy | Description |
|-----------|----------|-------------|
| `is_monorepo == false` | `single` | Анализировать весь проект как один модуль |
| Monorepo, один язык, ≤5 модулей | `per-module-with-shared-context` | Анализировать каждый модуль, шарить общий контекст |
| Monorepo, разные языки | `per-module` | Независимый анализ каждого модуля |
| Monorepo, >10 модулей | `per-module` | Слишком большой для shared context |

```
if !is_monorepo:
    strategy = "single"
    analysis_targets = [state.validate.path]
elif all_same_language AND module_count <= 5:
    strategy = "per-module-with-shared-context"
    analysis_targets = [module.path for module in modules]
else:
    strategy = "per-module"
    analysis_targets = [module.path for module in modules]
```
